# Routing traffic based on HTTP Headers `end-user` equals `jason` to reviews-v2



## Create Virtual Service for Routing
```shell
# kubectl apply -f samples/bookinfo/networking/virtual-service-reviews-test-v2.yaml -n bookinfo

```
## Show Virtual Service（Routing Rules）

```yaml
apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: reviews
spec:
  hosts:
    - reviews
  http:
  - match:
    - headers:
        end-user:
          exact: jason
    route:
    - destination:
        host: reviews
        subset: v2
  - route:
    - destination:
        host: reviews
        subset: v1

```

## Login /productpage in browser as `jason`

All traffic from `jason` route to black-star reviews subset (v2)
